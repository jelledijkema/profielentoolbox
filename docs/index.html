Werkwijze

Van GEF-bestand
tot FME-feature.

Geen handmatige conversie meer. De toolbox verwerkt het volledige sonderingsprofiel inclusief georeferentie-attributen, klaar voor directe import.

01 —

GEF aanleveren

Geef het pad naar het .gef bestand op. De toolbox leest de header en meetreeks automatisch uit.

02 —

Verwerken & valideren

Coordinaatstelsels worden herkend, dieptewaarden geconverteerd en output gevalideerd tegen het dataschema.

03 —

Inlezen in FME

De PythonCaller importeert de module en retourneert gestructureerde feature-lijsten, direct bruikbaar in de workflow.

Vroeger

PythonCaller als black box

  • ✗ Scripts verstopt in .fmw bestanden
  • ✗ Geen versiehistorie, geen git
  • ✗ Debuggen via logregels
  • ✗ Copy-paste tussen projecten
  • ✗ Testen niet mogelijk buiten FME

Nu

Een geïnstalleerd Python-pakket

  • ✓ Volledige IDE-ondersteuning (bijv. VSCode)
  • ✓ Git-history met betekenisvolle commits
  • ✓ Unit tests draaien in CI/CD
  • ✓ Herbruikbaar via pip install
  • ✓ PythonCaller als dunne wrapper

Integratie

Drie regels code.

De PythonCaller blijft een dunne wrapper. Alle logica leeft in het pakket — getest, gedocumenteerd, versioned.

# In je FME PythonCaller:
from profielentoolbox.extract_gef import parse_gef

profile = parse_gef("pad/naar/bestand.gef")
records = profile.to_records() # lijst van dicts per dieptemeting

# Elke record bevat: x, y, z, coordinate_system, + meetreeks
